### P1: GitHub OAuth state is not bound to the initiating browser

References:

- `backend/api-server/src/auth_handlers.rs:172`
- `backend/api-server/src/auth_handlers.rs:190`
- `backend/api-server/src/auth_handlers.rs:225`
- `backend/shared/src/db/sessions.rs:216`
- `backend/shared/src/db/sessions.rs:236`
- `frontends/web/src/components/creator/CreatorOAuthCallback.tsx:25`

Failure mode: any attacker can start OAuth, obtain a valid `state` and callback `code`, then trick a creator into opening `/creator/oauth/callback?code=...&state=...`.
The callback consumes only the globally stored `state_hash` and then issues a creator session for the attacker's GitHub identity in the victim's browser.

Fix direction: issue a short-lived HttpOnly/SameSite OAuth nonce cookie on `/api/auth/github/login`, store its hash with the OAuth state row, and require both the request `state` and the matching cookie before consuming the row.
Expire the nonce cookie on successful callback.
Remove the raw `state` from `GithubOauthLoginResponse` because the frontend only needs the authorization URL.

### P1: Official private bundles can be exposed through permissive runner scratch permissions

References:

- `backend/shared/src/runner/execution.rs:57`
- `backend/shared/src/runner/execution.rs:61`
- `backend/shared/src/runner/execution.rs:62`
- `backend/shared/src/runner/run_io.rs:45`
- `backend/shared/src/runner/run_io.rs:80`
- `backend/shared/src/config.rs:967`
- `ops/src/check_dgx_spark_profile.rs:206`

Failure mode: official evaluations copy the full private bundle into the per-attempt scratch tree and then make it container-readable by adding other-read/traverse bits.
Writable runner trees similarly get other-writable bits.
If the runner runtime root is missing or traversable by other host users, local host users or processes can inspect private benchmark data while the job runs.

Fix direction: production workers must require a configured service-owned runtime root with non-traversable permissions.
Create transient evaluation artifact roots and per-attempt roots as private host directories before broadening child permissions for Docker bind compatibility.
DGX profile checks must fail when runtime root permissions are too broad.
