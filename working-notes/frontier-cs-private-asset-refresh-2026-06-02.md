## Adapter Categories

- Generic separated stdio checker overlays.
- Generic separated file/path-backed checker overlays.
- Generic piped-stdio file-backed interactive session overlays.
- Exact custom inline/session adapters:
  - permutation reconstruction
  - uniform cave explorer
  - moving mole tree
  - treasure packing
  - world map
- Runtime-random official session adapters:
  - duplicate-position search
  - limited-shuffle restore
  - signed-rooted tree
  - treasure-hunt choices

Runtime-random official sessions keep public validation deterministic. Official
`private-benchmark/session.json` files contain public case parameters and
random-policy metadata only; hidden arrays, signs, seeds, and answers are
generated by the trusted evaluator at runtime.

## Commands Used

```bash
cargo run -p agentics-ops --bin agentics-refresh-frontier-cs-private-assets -- --dry-run
sudo just storage::backup-up
just storage::refresh-frontier-cs-private-assets --confirm-overwrite
sudo env PATH="$PATH" HOME="$HOME" CARGO_HOME="${CARGO_HOME:-$HOME/.cargo}" RUSTUP_HOME="${RUSTUP_HOME:-$HOME/.rustup}" just rehearsal::up
sudo env PATH="$PATH" HOME="$HOME" CARGO_HOME="${CARGO_HOME:-$HOME/.cargo}" RUSTUP_HOME="${RUSTUP_HOME:-$HOME/.rustup}" just rehearsal::restore-private-bundles --overwrite
```

The rehearsal restore copied all objects visible in the persistent backup store,
not only the refreshed 162, because the restore command is currently a broad
backup-store copy. The refreshed 162 destination keys were present under
`rehearsal/private-bundle-backups/<challenge_name>/official-runs.zip`.

## Unresolved Follow-Ups

- Consider adding a batch manifest or source-prefix filter so rehearsal restore
  can restore exactly a named refresh batch rather than all backup objects.
- Split `ops/src/frontier_cs_private_assets.rs` into adapter, backup, ZIP, and
  candidate modules before extending the refresh tool further; the first batch
  implementation is intentionally focused but already above the preferred
  source-file size threshold.
- Make the rehearsal purge or startup path more ergonomic for runner-daemon
  image reloading. Production-profile worker probes fail closed when the
  dedicated runner daemon lacks `busybox:1.36`; official CPU runs also need the
  digest-pinned CPU runner image loaded or pullable.
